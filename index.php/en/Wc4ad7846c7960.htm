The OMV metadata schema is formally represented as ontology and is designed modularly: OMV distinguishes between the OMV Core and various OMV Extensions. The core captures information which is expected to be relevant to the majority of ontology reuse settings, while the extensions allow ontology developers and users to specify task/application-specific ontology-related information (e.g. mappings, ontology evaluation, ontology changes etc,).
